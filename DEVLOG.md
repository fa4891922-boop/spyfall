## Стек / структура
- Бэкенд: `server.js` (Node.js + Express + Socket.IO), раздаёт клиент из `public/`.
- Клиент (рабочий): `public/index.html`, `public/app.js`, `public/styles.css`.
- Локации/роли/подсказки: `locations.js`.
- Деплой бэкенда: Render (`spyfall-wp86.onrender.com`).
- ВНИМАНИЕ: в корне есть отдельный `index.html` (React/Babel, GitHub Pages) — он несовместим
  с протоколом `server.js`. Рабочая версия — в `public/`.

## Журнал изменений

### Сессия 1
- Подсказки шпиону: у каждой локации одна тонкая подсказка (`hint`); шпион видит только её.
- Смена никнейма прямо в лобби (`changeName` + UI).
- Кик игроков из лобби (был ранее) — подтверждён рабочим.
- Расширен список тем-вопросов для шпиона; починен баг `TURN_SECONDS` на клиенте.
- Добавлен поиск локаций для шпиона (панель угадывания + общий список).

### Сессия 2
- Голосование можно запускать в любой момент, сколько угодно раз (новое заменяет текущее).
- Добавлена отмена голосования (кнопка + событие `cancelVote`/`voteCancelled`).
- Оптимизация: таймер хода переведён с рассылки тиков каждые 500 мс на один `setTimeout`.

### Сессия 3
- Полная замена набора локаций: старые удалены, добавлено 55 совершенно новых.
- Логика выбора локации — «мешок без повторов»: каждая выпадает 1 раз за круг,
  при новом круге колода перемешивается (последняя не повторяется сразу).

### Сессия 4
- Отменить голосование может ТОЛЬКО его инициатор (проверка `initiatorId` на сервере
  + кнопка отмены скрыта у всех, кроме инициатора).
- Заведён этот DEVLOG со списком использованных локаций.
