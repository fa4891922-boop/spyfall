## Стек / структура
- Бэкенд: `server.js` (Node.js + Express + Socket.IO), раздаёт клиент из `public/`.
- Клиент (рабочий): `public/index.html`, `public/app.js`, `public/styles.css`.
- Локации/роли/подсказки: `locations.js`.
- Деплой бэкенда: Render (`spyfall-wp86.onrender.com`).
- ВНИМАНИЕ: в корне есть отдельный `index.html` (React/Babel, GitHub Pages) — он несовместим
  с протоколом `server.js`. Рабочая версия — в `public/`.

## Журнал изменений

### Сессия 5 — Логирование и модальное подтверждение (v3.1)
- Добавлена структурированная сквозная система логирования (объект `LOG` на клиенте, функции `log`/`logErr` на сервере) с таймстампами, тегами и гибким форматированием данных для событий HTTP, сокетов, реконнекта, игрового процесса и ошибок.
- Добавлена поддержка кастомных заголовков для модального подтверждения `showConfirm` (параметр `title` с полной обратной совместимостью) и встроенная валидация текста перед отображением модалки (`LOG.assert`).
- Добавлены новые стили для элемента надзаголовка модального окна (`.confirm-modal .eyebrow`).

### Сессия 3 — большое упрощение (v3.0)
- Убраны роли: игрок видит только локацию или «ТЫ ШПИОН» (без профессий/ролей).
- Убран встроенный чат — общение теперь только в Discord.
- Убраны боты, спец-роли (детектив/напарник шпиона), голосовые сообщения, Redis-адаптер, i18n.
- Упрощён таймер: один общий таймер раунда (а не походовые ходы). Событие `timeUp` по окончании.
- Новый UI: крупный круговой таймер (SVG) с анимацией «warning/danger», счётчик раундов,
  таблица лидеров с расширенной статистикой (сколько раз был шпионом, побед).
- Шрифты Oswald + Manrope, тёмная шпионская палитра, адаптив + reduced-motion.
- `package.json` обновлён до v3.0.0, убраны лишние зависимости (redis, rate-limit, socket.io-client).
- Механика сохранена: голосование (единогласное), угадывание локации шпионом, reconnect, наборы локаций.

### Сессия 1
- Подсказки шпиону: у каждой локации одна тонкая подсказка (`hint`); шпион видит только её.
- Смена никнейма прямо в лобби (`changeName` + UI).
- Кик игроков из лобби (был ранее) — подтверждён рабочим.
- Расширен список тем-вопросов для шпиона; починен баг `TURN_SECONDS` на клиенте.
- Добавлен поиск локаций для шпиона (панель угадывания + общий список).

### Сессия 2
- Голосование можно запускать в любой момент, сколько угодно раз (новое заменяет текущее).
- Добавлена отмена голосования (кнопка + событие `cancelVote`/`voteCancelled`).
- Оптимизация: таймер хода переведён с рассылки тиков каждые 500 мс на один `setTimeout`.

### Сессия 3

  при новом круге колода перемешивается (последняя не повторяется сразу).

### Сессия 4

## v2.0 — Реализация полного плана развития (Фазы 1–5)

### Фаза 1 — Стабилизация
- Классический набор локаций вынесен в `data/locations/classic.json`; `locations.js` реэкспортирует его для обратной совместимости.
- Автоочистка пустых комнат раз в 30 мин (idle > 10 мин) — устранена утечка памяти.
- Лимит активных комнат (50) с вытеснением самой старой неактивной.
- Reconnect: `sessionId` в localStorage, событие `resume`, перенос игрока на новый `socket.id` с восстановлением роли и таймера.
- Эндпоинт `/stats` (комнаты, игроки, uptime, аналитика). Шеринг ссылкой `?room=XXXX` + авто-джойн.

### Фаза 2 — UX & Игровой опыт
- Мобильная адаптация: media-запросы, `min-height: 44px` на кнопках, `touch-action: manipulation`.
- Анимации: pop для карточек роли, pulse для таймера < 10 сек, плавные переходы экранов.
- Звуковые эффекты на Web Audio API (без файлов): начало/конец хода, голосование, разоблачение, тики.
- Гибкий таймер: длительность раунда и время на ход (15/30/45/60) выбираются в лобби.
- Режим зрителя: зритель видит локацию и шпиона, исключён из `speakingOrder` и голосований.
- Сохранение настроек в localStorage (имя, звук, язык, длительности, набор).

### Фаза 3 — Социальные фичи
- Текстовый чат: добавлен серверный обработчик `chatMessage` + быстрые фразы/эмодзи.
- Голосовые сообщения: добавлен обработчик `audioMessage` и отсутствовавшая кнопка записи `btnRecord` (исправлен критический баг — клиент падал на `null`).
- Таблица лидеров: `room.stats` (игры, разы шпионом, победы) — на экране итогов.
- Аватарки: цветной круг + первая буква по хешу имени.
- Итоги раунда: раскрытие всех ролей и результатов голосования.

### Фаза 4 — Расширяемость
- Локации в JSON + API `/api/locations` и `/api/locations/:set`.
- Тематические наборы: Города, Фильмы, История, Профессии.
- i18n (RU/EN) с автоопределением по `navigator.language`, файлы `public/lang/`.
- Кастомные роли: Детектив (видит локацию среди ложных) и Напарник шпиона.

### Фаза 5 — Масштабирование
- Опциональный Redis-адаптер (`REDIS_URL`) с in-memory fallback.
- Защита от спама: rate-limit на сокет-события и HTTP (`express-rate-limit`).
- Аналитика: `totalGames`, `totalPlayers` (по уникальным сессиям), популярные локации в `/stats`.
- Боты для заполнения комнаты (автоответы в чат, случайное голосование).
